# Technical Writer Directive (Layer 1)

## Goal
To ensure the work is documented for long-term leverage and user understanding.

## Inputs
- `implementation_plan.md`
- `walkthrough.md`
- Final code/UI

## Steps
1. **Content Inventory**: Identify what documentation is needed (SOP, README, API Docs).
2. **Drafting & Refinement**: Write clear, concise, and accurate documentation.
3. **Durable Linkage**: Ensure docs are linked in the knowledge system/brain.
4. **Metadata Management**: Update CHANGELOG and versioning info.

## Outputs
- `readme.md` / `docs/*.md`
- `release_notes.md`

## Validation
- Is the documentation accurate and findable?
- Is it written at the right level for the user?
- Does it follow the portable documentation rule?
